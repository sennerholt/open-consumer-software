# Contributing

Open Consumer Software is published here as a v0.1 draft design standard and portable skill package. Contributions are welcome when they clarify the doctrine, improve agent usability, or make the package easier to evaluate and adopt.

## Contribution Principles

- Preserve `README.md` as the canonical human-readable OCS reference.
- Preserve `SKILL.md` as the agent-readable operational instruction file.
- Keep the package portable across agent environments.
- Avoid adding platform-specific setup as a requirement for use.
- Treat consumer sovereignty, situated knowledge, durable rails, independent skins, shared state, and rails-enforced policy as core concepts.
- Distinguish clearly between OCS doctrine, implementation guidance, examples, and future governance questions.

## Good Contributions

- clearer definitions or qualification criteria
- better examples, non-examples, or borderline cases
- stronger operational guidance for agents
- corrections to inconsistent terminology or status labels
- packaging improvements that do not make the package harder to use as-is

## Changes That Need Extra Care

- changes to the canonical definition
- changes to normative `MUST`, `SHOULD`, or `MAY` requirements
- changes that narrow or weaken consumer sovereignty
- changes that make OCS depend on a specific vendor, agent runtime, protocol, or platform
- claims that a product is OCS-Native or OCS-Complete without enough supporting criteria

## Pull Requests

Pull requests should explain:

- what problem the change solves
- whether it changes doctrine, packaging, examples, or agent behavior
- whether `README.md` and `SKILL.md` remain aligned

By contributing, you agree that your contribution will be licensed under the repository license.
