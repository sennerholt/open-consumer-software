# Governance

Open Consumer Software is a v0.1 draft design standard and portable skill package.

## Project Status

OCS is usable as a design framework and agent skill package. It is not yet a formal certification program, protocol specification, standards body, or conformance authority.

## Decision Making

The project maintainer is responsible for final decisions on:

- canonical terminology
- normative requirements
- maturity model changes
- package structure
- release timing

Contributions and proposals are welcome through GitHub issues and pull requests. Changes that affect the canonical definition, normative requirements, or consumer sovereignty require extra review.

## Compatibility Expectations

The package should remain usable as a directory containing `README.md` and `SKILL.md`. Consumers should not need to edit repository files before using it.

## Future Governance

If OCS grows into a broader standard, future governance should define:

- conformance claims
- certification or trust labels
- compatibility policies
- decision rights
- maintainer roles
- appeal and dispute processes
