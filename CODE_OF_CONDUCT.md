# Code of Conduct

The Open Consumer Software project develops OCS as a serious design standard and portable agent skill package.

Participants are expected to:

- engage in good faith
- criticize ideas, not people
- keep discussion relevant to OCS doctrine, examples, packaging, and implementation
- make room for accessibility, consumer sovereignty, and situated user needs as legitimate design concerns
- avoid harassment, personal attacks, discriminatory language, and bad-faith disruption

Maintainers may moderate, close, or remove issues, pull requests, comments, or participation that undermine productive work on the project.

If you see behavior that should be addressed, open an issue if public discussion is appropriate. If the project later adds private reporting channels, this file should be updated.
